## Findings

### ⚠️ Finding 1 — "How it works" section missing from landing

**Guest landing** shows only: tagline ("Practical Japanese for real life"), auth form (Google + email/password), and "Try it free" guest link. No "How it works" section exists.  
The nav has a "How to use" link (→ `/how-to`) which is where that content lives.  
**Action:** Update Flow A checklist to remove "How it works" check on landing, or add a check that the How to use nav link exists.

### ⚠️ Finding 2 — Flashcard "Reveal" button stays visible after revealing

After clicking "Reveal", the button label and visibility don't change. The card back is shown (reading + translation), but "Reveal" remains as if it weren't clicked. Could be intentional (re-reveal if you look away), but could also be a missing state toggle.  
**Action:** Check whether Reveal should transition to a disabled or hidden state after first click.

### ⚠️ Finding 3 — Kana wrong-answer correction state not confirmed

When a wrong romaji choice is selected, the counter advances to the next kana without visible correction text in the page body. If there's a colour-only correct/wrong indicator, it won't surface in text extraction.  
**Action:** Manual visual check — does a wrong answer show a red highlight or "X — the correct answer was su" before advancing? If not, that's a real UX gap.

### ⚠️ Finding 4 — Conversation header shows N4 after N5 selected

Steps: conversation page → N5 radio/button clicked → "Start conversation" → conversation header reads "Hana · N4".  
Expected: "Hana · N5".  
**This is the highest-priority finding.** Either the level selector click isn't registering, the N4/N5 buttons toggle rather than select, or the display logic has a mapping error.  
**Action:** Investigate `ConversationPage` level selector state — confirm selected level is passed correctly to the session start call and reflected in the header.
